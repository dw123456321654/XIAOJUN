## 文档目的

本文档定义了2048游戏的完整产品需求，明确功能范围、用户故事、验收标准，作为后续规划、设计、开发的依据。

---

## 需求背景

### 项目概述
2048是一款经典的数字合并益智游戏，玩家通过滑动屏幕或按键移动数字方块，相同数字碰撞合并，目标是合成出2048数字。

### 项目目标
- 为用户提供一个经典、流畅的2048游戏体验
- 响应式设计，支持桌面端和移动端
- 代码简洁，适合新手学习和扩展

### 目标用户
- 喜欢益智游戏的休闲玩家
- 想要放松的上班族
- 学习HTML/JavaScript的开发新手

## 用户故事

### US-001：开始游戏
作为玩家，我希望打开游戏后立即看到准备好的游戏界面，这样我可以快速开始游戏。

**验收标准**：页面加载后，2秒内显示完整的游戏界面，包含网格、分数、按钮。

### US-002：移动数字
作为玩家，我希望按方向键或滑动屏幕时数字能正确移动，这样我可以控制游戏进程。

**验收标准**：按任意方向键，所有数字朝该方向移动到最远位置。

### US-003：合并数字
作为玩家，我希望相同数字碰撞时能合并成更大的数字，这样我可以追求更高分。

**验收标准**：两个相同数字碰撞时，合并成一个两倍数字，得分增加。

### US-004：记录最高分
作为玩家，我希望我的最高分被保存，这样我可以挑战自己。

**验收标准**：刷新页面后，最高分仍然显示正确。

### US-005：重新开始
作为玩家，我希望可以随时重新开始，这样我可以重新挑战。

**验收标准**：点击"重新开始"按钮，游戏立即重置。

## 功能需求

### REQ-001：游戏初始化
**优先级：** P0（必须）

**功能描述：**
游戏启动时显示4x4空白网格，随机生成2个数字（2或4）在不同位置，显示初始分数为0和最高分（从localStorage读取）。

**输入：** 无
**输出：** 游戏初始状态

**详细说明：**
- 网格尺寸：4行4列，共16个格子
- 初始生成2个数字：90%概率为2，10%概率为4
- 两个数字必须在不同位置
- 初始分数：0
- 最高分：从localStorage读取键"game2048_best"，不存在则为0

**边界条件：**
- localStorage不可用时，最高分默认为0

**异常处理：**
- localStorage读取失败时，静默使用默认值0，不报错

**验收标准：**
1. 页面加载完成后，网格中正好有2个数字，其余14个位置为空
2. 两个数字都是2或4
3. 两个数字在不同位置
4. 分数显示为0
5. 最高分显示正确值（如果有历史记录）

---

### REQ-002：数字移动
**优先级：** P0（必须）

**功能描述：**
玩家按键盘方向键（ArrowUp、ArrowDown、ArrowLeft、ArrowRight）或在触摸屏上滑动，所有数字朝指定方向移动到最远空白位置。

**输入：**
- 键盘事件：ArrowUp、ArrowDown、ArrowLeft、ArrowRight
- 触摸事件：touchstart、touchend（计算滑动方向）

**输出：** 网格状态更新

**详细说明：**
- 向上移动：每列数字向上移动，数字靠拢到最上方
- 向下移动：每列数字向下移动，数字靠拢到最下方
- 向左移动：每行数字向左移动，数字靠拢到最左侧
- 向右移动：每行数字向右移动，数字靠拢到最右侧
- 触摸滑动：计算touchstart和touchend的坐标差，判断滑动方向（最小滑动距离50px）

**边界条件：**
- 数字已在最边缘时，该方向移动无效
- 滑动距离小于50px时不触发移动

**异常处理：**
- 无（键盘和触摸事件由浏览器原生支持）

**验收标准：**
1. 按上键：所有数字尽可能向上移动
2. 按下键：所有数字尽可能向下移动
3. 按左键：所有数字尽可能向左移动
4. 按右键：所有数字尽可能向右移动
5. 向上滑动（距离≥50px）：触发向上移动
6. 向下滑动（距离≥50px）：触发向下移动
7. 向左滑动（距离≥50px）：触发向左移动
8. 向右滑动（距离≥50px）：触发向右移动

---

### REQ-003：数字合并
**优先级：** P0（必须）

**功能描述：**
相同数字在移动过程中碰撞时，合并成一个更大的数字（值为原数字的两倍），得分增加合并后的数值。

**输入：** 当前网格状态
**输出：** 更新后的网格状态、增加的分数

**详细说明：**
- 两个相同数字碰撞：合并成新数字，值为原数字×2
- 三个相同数字排列：前两个先合并，第三个单独
- 四个相同数字排列：先合并前两个，再合并后两个
- 每次移动中，每个数字最多参与一次合并
- 合并后的新数字放在碰撞位置（靠近移动方向的位置）
- 得分增加：合并后的数字值（如合并两个4得8分）

**边界条件：**
- 数字为0（空格）不参与合并
- 不同数字碰撞不合并

**异常处理：**
- 无

**验收标准：**
1. 两个2碰撞 → 变成一个4，得分+4
2. 两个4碰撞 → 变成一个8，得分+8
3. 三个2排列：前两个2合并成一个4，第三个2单独
4. 四个2排列：合并成两个4（分别在前两个和后两个位置）
5. 数字2和数字4碰撞 → 不合并，各自移动到相邻位置

---

### REQ-004：新数字生成
**优先级：** P0（必须）

**功能描述：**
每次有效移动后，在随机空白位置生成一个新数字（2或4）。

**输入：** 当前网格状态（移动后）
**输出：** 更新后的网格状态（含新生成的数字）

**详细说明：**
- 仅在有效移动后生成新数字（至少有一个数字移动或合并）
- 在所有空白位置中随机选择一个位置
- 生成数字值：90%概率为2，10%概率为4
- 新数字有出现动画（从scale 0放大到scale 1）

**边界条件：**
- 网格已满（无空白位置）时不生成新数字

**异常处理：**
- 无

**验收标准：**
1. 每次有效移动后，网格中新增一个数字
2. 新数字在之前为空的位置
3. 新数字是2（90%概率）或4（10%概率）
4. 新数字有出现动画

---

### REQ-005：无效移动处理
**优先级：** P0（必须）

**功能描述：**
当移动无效（所有数字已靠边或没有可移动空间）时，不生成新数字，不更新游戏状态。

**输入：** 用户操作（按键或滑动）
**输出：** 无变化

**详细说明：**
- 判断移动是否有效：移动后是否有数字位置变化或数字合并
- 无效移动包括：
  - 所有数字已在最左边时，按左键
  - 所有数字已在最右边时，按右键
  - 所有数字已在最上边时，按上键
  - 所有数字已在最下边时，按下键
  - 网格已满且没有相邻相同数字时，任意方向移动
- 无效移动时：
  - 网格状态不变
  - 分数不变
  - 不生成新数字

**边界条件：**
- 无

**异常处理：**
- 无

**验收标准：**
1. 所有数字已在最左边时，按左键无效（网格、分数不变，不生成新数字）
2. 所有数字已在最右边时，按右键无效
3. 所有数字已在最上边时，按上键无效
4. 所有数字已在最下边时，按下键无效
5. 网格已满且无相邻相同数字时，任意方向移动无效

---

### REQ-006：游戏结束判断
**优先级：** P0（必须）

**功能描述：**
检测游戏是否结束（所有格子填满且无法再移动），游戏结束时显示弹窗并阻止进一步操作。

**输入：** 当前网格状态
**输出：** 游戏结束状态（true/false）

**详细说明：**
- 每次移动后检测游戏是否结束
- 游戏结束条件：
  - 16个格子全满（无空白位置）
  - 且没有相邻相同数字（上下左右相邻的两个数字相同）
- 游戏结束时：
  - 显示游戏结束弹窗
  - 弹窗显示最终得分
  - 弹窗包含"再来一局"按钮
  - 阻止玩家继续移动（忽略键盘和触摸事件）

**边界条件：**
- 网格未满时不判断游戏结束
- 有空白位置时不判断游戏结束

**异常处理：**
- 无

**验收标准：**
1. 16个格子全满且无相邻相同数字时 → 游戏结束
2. 游戏结束弹窗显示，包含最终得分
3. 弹窗上有"再来一局"按钮
4. 游戏结束状态下，玩家无法移动数字
5. 点击"再来一局"按钮，游戏重新开始

---

### REQ-007：重新开始
**优先级：** P0（必须）

**功能描述：**
玩家点击"重新开始"按钮或在游戏结束弹窗中点击"再来一局"时，游戏重置到初始状态。

**输入：** 点击事件
**输出：** 游戏重置到初始状态

**详细说明：**
- 清空所有格子（所有位置数字为0）
- 重置当前分数为0
- 重新生成2个初始数字（2或4，在不同位置）
- 隐藏游戏结束弹窗（如果显示）
- 最高分保持不变（不重置）

**边界条件：**
- 无

**异常处理：**
- 无

**验收标准：**
1. 点击"重新开始"按钮后，网格清空
2. 2个新数字出现在不同位置
3. 分数归零
4. 最高分保持不变
5. 游戏结束弹窗隐藏

---

### REQ-008：最高分保存
**优先级：** P0（必须）

**功能描述：**
游戏过程中实时更新最高分，使用localStorage保存，刷新页面后能正确读取。

**输入：** 当前分数
**输出：** 保存的最高分

**详细说明：**
- localStorage键名："game2048_best"
- 每次得分增加后，检查当前分数是否超过最高分
- 当前分数 > 最高分时，更新最高分并保存到localStorage
- 页面加载时，从localStorage读取最高分
- localStorage不可用时，使用内存变量存储（不报错）

**边界条件：**
- localStorage不存在或被禁用时，最高分保存在内存变量中

**异常处理：**
- localStorage写入失败时，静默失败，不影响游戏进行
- localStorage读取失败时，使用默认值0

**验收标准：**
1. 得分超过当前最高分时，最高分更新
2. 刷新页面，最高分仍显示正确
3. 最高分只增不减（除非手动清除localStorage）
4. localStorage不可用时，游戏正常运行，最高分临时保存在内存中

## 非功能需求

### NFR-001：性能
**描述：** 游戏运行流畅，无明显卡顿

**验收标准：**
- 移动动画时长≤150ms
- 数字合并动画时长≤150ms
- CPU占用率在普通设备上<10%

---

### NFR-002：兼容性
**描述：** 支持主流浏览器

**验收标准：**
- Chrome/Edge：最新版本及前3个版本
- Firefox：最新版本及前3个版本
- Safari：最新版本
- 移动浏览器：iOS Safari、Chrome Mobile

---

### NFR-003：响应式设计
**描述：** 适配不同屏幕尺寸

**验收标准：**
- 桌面端（≥1024px）：固定尺寸居中显示
- 平板端（768px-1023px）：自适应宽度
- 手机端（<768px）：全屏显示，支持触摸滑动
- 网格尺寸在手机端自动缩放

---

### NFR-004：可访问性
**描述：** 基本可访问性支持

**验收标准：**
- 键盘可操作所有功能
- 重新开始按钮可Tab选中
- 颜色对比度符合WCAG AA标准

## UI/UX需求

### UI-001：配色方案
采用经典2048配色：
- 背景色：#faf8ef（米白色）
- 网格背景：#bbada0（浅棕色）
- 空格背景：#cdc1b4（浅灰棕色）
- 文字颜色：#776e65（深棕色）

### UI-002：数字方块样式
根据数字大小显示不同背景色和文字大小：
- 2：#eee4da背景，#776e65文字
- 4：#ede0c8背景，#776e65文字
- 8：#f2b179背景，#f9f6f2文字
- 16：#f59563背景，#f9f6f2文字
- 32：#f67c5f背景，#f9f6f2文字
- 64：#f65e3b背景，#f9f6f2文字
- 128：#edcf72背景，#f9f6f2文字
- 256：#edcc61背景，#f9f6f2文字
- 512：#edc850背景，#f9f6f2文字
- 1024：#edc53f背景，#f9f6f2文字
- 2048：#edc22e背景，#f9f6f2文字
- >2048：#3c3a32背景，#f9f6f2文字

### UI-003：动画效果
- 移动动画：平滑滑动（CSS transition，0.15s）
- 合并动画：轻微放大再缩小（scale 1.1 → 1.0，0.15s）
- 生成动画：从0放大到正常大小（scale 0 → 1.0，0.15s）

### UI-004：布局结构
```
标题区："2048"
分数区：[分数: 128] [最高分: 4096]
游戏网格：4x4
按钮区：[重新开始]
提示区："↑↓←→ 或滑动移动"
```

## 优先级总结

### P0（必须）- MVP范围
- REQ-001：游戏初始化
- REQ-002：数字移动
- REQ-003：数字合并
- REQ-004：新数字生成
- REQ-005：无效移动处理
- REQ-006：游戏结束判断
- REQ-007：重新开始
- REQ-008：最高分保存

### P1（重要）- 第二迭代
- 撤销功能（可撤销最近3步操作）
- 音效（移动、合并、游戏结束）
- 星级评价（基于最终得分）
- 分享功能（分享到社交媒体）

### P2（一般）- 第三迭代
- 皮肤系统（不同主题配色）
- 排行榜（本地排行榜或在线排行榜）
- 教程模式（新手引导）
- 成就系统

### P3（可选）- 后续迭代
- 多难度（3x3、5x5网格）
- 计时模式
- 挑战模式（特定开局）
- 数据统计（总游戏次数、胜率等）

## 依赖项

- 浏览器：支持localStorage、CSS3动画、触摸事件
- 无外部依赖（纯HTML/CSS/JavaScript）

## 风险与约束

### 风险
- 移动端触摸滑动可能误触发浏览器滚动 → 使用preventDefault阻止默认行为
- localStorage可能被禁用 → 优雅降级，使用内存变量存储
- 动画性能在低端设备可能卡顿 → 禁用动画选项

### 约束
- 必须为纯HTML/CSS/JavaScript实现，不依赖第三方库
- 单文件部署（HTML、CSS、JS可以合并为一个文件）
- 页面大小<100KB（压缩后）
