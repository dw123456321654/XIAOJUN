## 2. 文件命名规范

### 2.1 文档文件命名

**格式：** `类型_主题.md`

**示例：**
- `需求文档.md` - 产品需求
- `测试报告-逻辑错误.md` - 测试报告
- `修复报告.md` - 修复报告
- `诊断指南.md` - 诊断指南
- `开发日志.md` - 开发日志
- `2026-03-16.md` - 按日期的日志

**规则：**
- 使用中文命名，清晰描述文件内容
- 类型前缀：`需求文档_`、`测试报告_`、`修复报告_`、`开发日志_`
- 特殊报告：`诊断指南_`、`故障排除_`
- 日期格式：`YYYY-MM-DD.md`

### 2.2 代码文件命名

**格式：** 遵循编程语言命名规范

**Python：** `snake_case.py`
- 示例：`test_main.py`、`utils.py`、`api_client.py`

**JavaScript：** `camelCase.js` 或 `kebab-case.js`
- 示例：`testLogic.js`、`main.js`、`api-client.js`

**配置文件：** `config.ini`、`.env`
- 示例：`app.config`、`database.env`

### 2.3 测试文件命名

**格式：** `test_功能名.js` 或 `功能名_test.js`

**示例：**
- `test_move_logic.js` - 移动逻辑测试
- `test_merge_array.js` - 合并数组测试
- `test_generate_number.js` - 生成数字测试

**规则：**
- 以 `test_` 或 `_test` 为前缀/后缀
- 使用英文命名，符合编程语言规范
- 文件名描述测试的功能或模块

### 2.4 临时文件命名

**格式：** `temp_描述.ext` 或 `tmp_描述.ext`

**示例：**
- `temp_test_data.json` - 临时测试数据
- `tmp_backup.zip` - 临时备份文件
- `temp_diagram.png` - 临时图表文件

**规则：**
- 以 `temp_` 或 `tmp_` 为前缀
- 描述文件用途或类型
- 临时文件可以随时删除

## 3. 文件归属规则

### 3.1 根目录文件

**必需文件：**
- `index.html` - Web项目主文件
- `README.md` - 项目说明

**可选文件：**
- `.gitignore` - Git 忽略文件

**禁止文件：**
- 临时文件（应放在 `temp/` 目录）
- 测试文件（应放在 `tests/` 目录）
- 报告文件（应放在 `reports/` 目录）
- 日志文件（应放在 `logs/` 目录）

### 3.2 docs/ 目录

**所有项目文档，按类型分文件夹：**

| 子目录 | 用途 | 文件类型 |
|--------|------|---------|
| `prd/` | 产品需求文档 | `.md` |
| `design/` | 设计文档 | `.md` |
| `project/` | 项目管理文档 | `.md` |
| `development/` | 开发相关文档 | `.md` |

**规则：**
- 所有文档使用 Markdown 格式
- 文件名使用中文命名
- 按类型分文件夹存储

### 3.3 logs/ 目录

**开发日志和运行日志：**

| 文件名 | 用途 | 更新频率 |
|--------|------|---------|
| `开发日志.md` | 开发过程记录 | 每日更新 |
| `YYYY-MM-DD.md` | 按日期归档的日志 | 每日创建 |

**规则：**
- 使用中文命名
- 开发日志记录每日进展
- 可按日期归档

### 3.4 reports/ 目录

**测试和修复报告：**

| 文件名 | 用途 | 文件类型 |
|--------|------|---------|
| `测试报告-*.md` | 测试报告 | `.md` |
| `修复报告.md` | 修复报告 | `.md` |
| `诊断指南.md` | 诊断指南 | `.md` |
| `性能报告.md` | 性能报告 | `.md` |

**规则：**
- 使用中文命名
- 格式：`类型_描述.md` 或 `类型_问题.md`
- 特殊报告：`诊断指南_`、`故障排除_`

### 3.5 tests/ 目录

**测试代码（临时，可删除）：**

| 文件名 | 用途 | 生命周期 |
|--------|------|---------|
| `test_*.js` 或 `*_test.js` | 测试代码 | 临时 |
| `test_*.py` 或 `*_test.py` | 测试代码 | 临时 |
| `模拟测试.js` | 模拟测试代码 | 临时 |

**规则：**
- 以 `test_` 或 `_test` 为前缀/后缀
- 使用英文命名，符合编程语言规范
- 测试完成后可以删除

**注意：** `tests/` 目录下的文件是临时的，可以随时删除，不影响项目运行。

### 3.6 temp/ 目录

**临时文件（随时可删除）：**

| 文件名 | 用途 | 生命周期 |
|--------|------|---------|
| `temp_*.ext` | 临时文件 | 随时删除 |
| `tmp_*.ext` | 临时文件 | 随时删除 |
| `backup_*.ext` | 备份文件 | 用完删除 |

**规则：**
- 以 `temp_` 或 `tmp_` 为前缀
- 描述文件用途或类型
- 随时删除，不提交到版本控制

**注意：** `temp/` 目录下的文件是临时的，可以随时删除，不影响项目运行。

## 4. 文件生命周期管理

### 4.1 永久文件（不删除）

**定义：** 项目运行所必需的文件，不删除。

**列表：**
- 根目录：`index.html`、`README.md`、`.gitignore`（可选）
- `docs/` 目录：所有文档
- `logs/` 目录：开发日志
- `src/` 目录：源代码（如适用）
- `config/` 目录：配置文件（如适用）

### 4.2 临时文件（随时可删除）

**定义：** 临时使用、可随时删除的文件。

**列表：**
- `tests/` 目录：测试代码
- `temp/` 目录：临时文件
- `reports/` 目录：已完成并可归档的测试报告

**注意：**
- `tests/` 和 `temp/` 目录可以随时删除
- `reports/` 目录中的报告可归档到 `docs/` 后删除

### 4.3 归档文件

**定义：** 不再活跃但需要保留的历史文件。

**归档位置：**
- `docs/archives/` - 归档的文档
- `logs/archives/` - 归档的日志

**归档规则：**
- 里程碑完成后归档 `reports/` 到 `docs/archives/`
- 项目完成后归档 `logs/` 中的按日期归档的日志
- 归档文件按日期或版本号组织

## 5. 文件管理最佳实践

### 5.1 文件命名

1. **文档使用中文命名**：清晰描述文件内容
2. **代码使用英文命名**：符合编程语言规范
3. **避免使用特殊字符**：空格、符号等
4. **使用短小精悍的文件名**：避免过长的文件名

### 5.2 文件组织

1. **按类型分目录**：`docs/`、`logs/`、`reports/` 等
2. **根目录保持简洁**：只保留必需文件
3. **临时文件及时清理**：定期删除 `tests/` 和 `temp/`
4. **定期归档历史文件**：不活跃文件归档到 `archives/`

### 5.3 版本控制

1. **提交到版本控制**：
   - `docs/`、`src/`、`config/`：必须提交
   - `logs/`：重要日志提交，临时日志不提交
   - `tests/` 和 `temp/`：不提交
   - `.gitignore`：忽略 `tests/` 和 `temp/`

2. **.gitignore 配置**：
```
tests/
temp/
*.log
*.tmp
temp_*
*.swp
.DS_Store
```

### 5.4 定期维护

1. **每周清理临时文件**：删除 `tests/` 和 `temp/`
2. **每月归档历史文件**：归档 `reports/` 和 `logs/` 中的旧文件
3. **每季度审查目录结构**：确保文件组织合理

### 7.2 清理脚本

**cleanup.py** - 清理临时文件
```python
import os
import shutil

def cleanup_temp_files(project_dir):
    # 删除 tests/ 目录
    tests_dir = os.path.join(project_dir, 'tests')
    if os.path.exists(tests_dir):
        shutil.rmtree(tests_dir)
        print(f'已删除: {tests_dir}')

    # 删除 temp/ 目录
    temp_dir = os.path.join(project_dir, 'temp')
    if os.path.exists(temp_dir):
        shutil.rmtree(temp_dir)
        print(f'已删除: {temp_dir}')

if __name__ == '__main__':
    cleanup_temp_files(r'D:/Projects/项目名称')
```
