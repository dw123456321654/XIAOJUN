## 风险清单

### R-001：技术风险 - 移动端触摸滑动误触发浏览器滚动

**风险描述：**
在移动端使用触摸滑动时，可能会误触发浏览器的默认滚动行为，影响游戏体验。

**影响范围：**
- 功能需求：REQ-002（数字移动 - 触摸事件）
- 非功能需求：NFR-003（响应式设计）

**风险等级：** 🟠 中等

**发生概率：** 高（70%）

**影响程度：** 中等（用户体验下降）

**应对措施：**
1. 使用 `event.preventDefault()` 阻止默认滚动行为
2. 仅在游戏区域内阻止滚动，不影响页面其他部分
3. 添加测试用例，验证各移动浏览器的兼容性

**责任人：** 开发人员
**时间节点：** 开发阶段（任务3.3）

---

### R-002：技术风险 - localStorage被禁用导致最高分丢失

**风险描述：**
浏览器localStorage可能被用户禁用或因隐私模式不可用，导致最高分无法保存。

**影响范围：**
- 功能需求：REQ-008（最高分保存）

**风险等级：** 🟡 低

**发生概率：** 中（30%）

**影响程度：** 低（游戏仍可玩，只是最高分不持久）

**应对措施：**
1. PRD已定义异常处理：localStorage不可用时，使用内存变量存储
2. 添加try-catch包裹localStorage操作
3. 在首次使用时检测localStorage是否可用
4. 可选：显示提示信息告知用户

**责任人：** 开发人员
**时间节点：** 开发阶段（任务3.2）

---

### R-003：性能风险 - 动画在低端设备卡顿

**风险描述：**
CSS动画在低端移动设备或旧版浏览器上可能卡顿，影响游戏体验。

**影响范围：**
- 非功能需求：NFR-001（性能）、NFR-003（响应式设计）

**风险等级：** 🟠 中等

**发生概率：** 中（40%）

**影响程度：** 中等（低端设备用户体验下降）

**应对措施：**
1. 优化动画性能：使用transform和opacity（GPU加速）
2. 动画时长控制在150ms以内
3. 可选：添加"关闭动画"选项，用户可自主选择
4. 使用requestAnimationFrame而非setInterval

**责任人：** 开发人员
**时间节点：** 开发阶段（任务3.4）

---

### R-004：兼容性风险 - 旧版浏览器不支持某些CSS特性

**风险描述：**
旧版浏览器（如IE11）可能不支持Flexbox、CSS Grid或某些CSS3特性。

**影响范围：**
- 非功能需求：NFR-002（兼容性）

**风险等级：** 🟢 低

**发生概率：** 低（10%）

**影响程度：** 低（目标用户使用现代浏览器）

**应对措施：**
1. PRD已明确兼容性要求：仅支持现代浏览器（最新版本及前3个版本）
2. 使用浏览器前缀（-webkit-、-moz-等）增强兼容性
3. 添加浏览器检测，不支持的浏览器显示提示信息
4. 使用Can I Use网站查询特性兼容性

**责任人：** 开发人员
**时间节点：** 测试阶段（任务4.2）

---

### R-005：进度风险 - 新手开发可能超出预估工时

**风险描述：**
开发人员为新手，可能对JavaScript某些概念（如闭包、异步）不熟悉，导致开发时间超出预估。

**影响范围：**
- 整体项目进度

**风险等级：** 🟠 中等

**发生概率：** 中（40%）

**影响程度：** 中等（延期1-2天）

**应对措施：**
1. 工作量评估已增加33%的缓冲时间
2. 建议开发前学习相关JavaScript基础知识（2-3小时）
3. 遇到技术难点及时向架构师求助
4. 关键任务设置检查点，及时调整计划

**责任人：** 项目经理、架构师、开发人员
**时间节点：** 全流程

---

### R-006：质量风险 - 边界情况处理不完善

**风险描述：**
游戏中存在多种边界情况（如四个相同数字排列、边缘移动等），可能被遗漏导致bug。

**影响范围：**
- 所有功能需求

**风险等级：** 🟠 中等

**发生概率：** 高（60%）

**影响程度：** 中等（游戏逻辑错误）

**应对措施：**
1. 测试人员设计全面的测试用例，覆盖所有边界情况
2. 开发人员在实现时主动思考边界情况
3. 代码Review时重点检查边界处理逻辑
4. 重点测试：四个相同数字合并、边缘移动、游戏结束判断

**责任人：** 开发人员、测试人员
**时间节点：** 测试阶段（任务4.1）

---

### R-007：需求风险 - 需求理解偏差

**风险描述：**
开发人员对需求的理解可能与产品经理不一致，导致实现偏差。

**影响范围：**
- 所有功能需求

**风险等级：** 🟠 中等

**发生概率：** 中（40%）

**影响程度：** 中等（返工修复）

**应对措施：**
1. 设计评审会上明确需求细节
2. 开发前开发人员与产品经理沟通确认
3. 开发过程中遇到疑问立即沟通
4. 验收标准必须明确可测试

**责任人：** 产品经理、开发人员、项目经理
**时间节点：** 开发前和开发中

---

### R-008：环境风险 - 缺少移动设备测试

**风险描述：**
可能缺少iOS/Android设备进行触摸操作测试，影响兼容性验证。

**影响范围：**
- 非功能需求：NFR-002（兼容性）、NFR-003（响应式设计）

**风险等级：** 🟢 低

**发生概率：** 中（50%）

**影响程度：** 低（可通过浏览器模拟器测试）

**应对措施：**
1. 优先使用浏览器开发者工具的设备模拟器
2. 如果有真实设备，优先在真实设备上测试
3. 记录已知设备兼容性问题
4. 可选：请朋友/同事帮忙测试

**责任人：** 测试人员
**时间节点：** 测试阶段（任务4.2）

---

### R-009：范围蔓延风险 - 功能需求不断增加

**风险描述：**
开发过程中可能不断新增功能需求（如音效、撤销、皮肤等），超出MVP范围。

**影响范围：**
- 整体项目进度和质量

**风险等级：** 🟠 中等

**发生概率：** 中（40%）

**影响程度：** 高（严重延期或质量下降）

**应对措施：**
1. PRD已明确MVP范围（8个P0功能）
2. 非P0功能严格控制在第二迭代
3. 需求变更需经过项目经理评估影响
4. 必要时与产品经理沟通，降低优先级或推迟到下一版本

**责任人：** 项目经理、产品经理
**时间节点：** 全流程

---

### R-010：交付风险 - 单文件部署可能导致代码组织混乱

**风险描述：**
PRD要求单文件部署，所有HTML、CSS、JS合并为一个文件，可能导致代码难以维护。

**影响范围：**
- 代码质量和可维护性

**风险等级：** 🟢 低

**发生概率：** 中（50%）

**影响程度：** 低（可以通过注释和结构化代码缓解）

**应对措施：**
1. 开发时先分离成独立文件（HTML、CSS、JS）
2. 代码结构清晰，模块化组织
3. 添加充分的注释说明各部分功能
4. 最后合并时保留清晰的分隔标记
5. 代码审查时检查代码组织结构

**责任人：** 开发人员、架构师
**时间节点：** 开发阶段

2. **R-006：边界情况处理不完善** - 🟠 中
   - 需充分测试，覆盖所有边界情况

3. **R-001：移动端触摸滑动误触发** - 🟠 中
   - 需正确处理触摸事件，避免误触发滚动

### 风险缓解建议

1. **加强沟通：** 开发前明确需求，开发中及时沟通
2. **充分测试：** 设计全面测试用例，覆盖边界情况
3. **控制范围：** 严格执行MVP范围，非P0功能推迟
4. **预留缓冲：** 工作量评估已增加33%缓冲时间
5. **代码审查：** 关键代码需经过审查，减少bug

### 成功关键

- **需求清晰：** 验收标准明确可测试
- **计划务实：** 工作量评估合理，有缓冲时间
- **沟通及时：** 遇到问题立即沟通解决
- **测试充分：** 覆盖所有功能和边界情况
- **控制范围：** 严格执行MVP，不盲目扩展
