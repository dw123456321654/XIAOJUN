### 1.4 Vue规范

#### 组件定义

```vue
<!-- ✅ 推荐：使用<script setup> -->
<script setup lang="ts">
import { ref, computed } from 'vue';

// 响应式数据
const count = ref(0);

// 计算属性
const doubleCount = computed(() => count.value * 2);

// 方法
const increment = () => {
  count.value++;
};
</script>

<!-- ❌ 避免：使用Options API（除非必要） -->
<script>
export default {
  data() {
    return { count: 0 }
  },
  computed: {
    doubleCount() {
      return this.count * 2
    }
  },
  methods: {
    increment() {
      this.count++
    }
  }
}
</script>
```

#### 模板语法

```vue
<!-- ✅ 推荐：使用简洁的模板语法 -->
<template>
  <div class="container">
    <h1>{{ title }}</h1>
    <p v-if="isVisible">Visible text</p>
    <button @click="handleClick">Click</button>
  </div>
</template>

<!-- ❌ 避免：复杂的模板逻辑 -->
<template>
  <div>
    <h1 v-text="title"></h1>
    <p v-show="isVisible">Visible text</p>
    <button v-on:click="handleClick">Click</button>
  </div>
</template>
```

## 3. Git提交规范

### 3.1 提交信息格式

```
<type>(<scope>): <subject>

<body>

<footer>
```

#### Type（类型）

| 类型 | 说明 |
|------|------|
| feat | 新功能 |
| fix | Bug修复 |
| docs | 文档更新 |
| style | 代码格式调整 |
| refactor | 重构代码 |
| test | 测试相关 |
| chore | 构建/工具链相关 |

#### 示例

```bash
# 新功能
feat(formula): add formula search functionality

# Bug修复
fix(search): fix fuzzy search algorithm

# 文档更新
docs(readme): update installation guide

# 重构
refactor(api): simplify API response format
```

### 3.2 提交频率

```bash
# ✅ 推荐：频繁提交，每次提交保持功能可运行
git add .
git commit -m "feat(formula): add formula detail page"
git push

# ❌ 避免：一次提交包含大量未验证的代码
git add .

## 8. 最佳实践

### 8.1 前端最佳实践

1. **组件化**：保持组件单一职责
2. **性能优化**：使用虚拟滚动、懒加载
3. **状态管理**：合理使用Pinia，避免过度使用全局状态
4. **错误处理**：完善的错误捕获和提示
5. **用户体验**：良好的加载状态和错误提示

### 8.2 后端最佳实践

1. **安全性**：输入验证、输出编码
2. **性能**：避免不必要的计算和内存分配
3. **错误处理**：使用Result类型，避免panic
4. **文档**：完善的函数文档注释
5. **测试**：充分的单元测试和集成测试
