### 2.3 数据模型

#### 公式数据模型

```typescript
interface Formula {
  id: string;                          // 公式ID
  name: string;                        // 公式名称
  category: {
    main: string;                      // 主分类
    sub: string;                       // 子分类
  };
  level: '初级' | '中级' | '高级';     // 难度
  popularity: number;                  // 热度 (1-5)
  version: 'common' | '365' | 'rare'; // 版本标签
  description: string;                 // 功能描述
  syntax: string;                      // 语法
  parameters: Parameter[];             // 参数列表
  examples: Example[];                 // 示例列表
  errors: ErrorCase[];                 // 错误案例
  usage_tips: string[];                // 使用建议
  warnings: string[];                  // 注意事项
  related: string[];                   // 相关公式
}

interface Parameter {
  name: string;                        // 参数名
  required: boolean;                   // 是否必需
  type: string;                        // 参数类型
  description: string;                 // 参数说明
}

interface Example {
  title: string;                       // 示例标题
  scenario: string;                    // 场景说明
  data: any[][];                       // 数据表格
  formula: string;                     // 公式
  result: string;                      // 结果
  explanation: string[];               // 详细说明
}

interface ErrorCase {
  title: string;                       // 错误标题
  formula: string;                     // 错误公式
  result: string;                      // 错误结果
  reason: string;                      // 错误原因
  fix: string;                         // 修正方案
}
```

#### 用户数据模型

```typescript
interface User {
  id: string;                          // 用户ID
  created_at: string;                  // 创建时间
  updated_at: string;                  // 更新时间
}

interface FavoriteFolder {
  id: string;                          // 收藏夹ID
  name: string;                        // 收藏夹名称
  formulas: string[];                  // 公式ID列表
  created_at: string;                  // 创建时间
  updated_at: string;                  // 更新时间
}

interface LearningProgress {
  learned: string[];                  // 已学习公式
  total: number;                       // 总数
  categories: CategoryProgress[];      // 分类进度
  achievements: Achievement[];          // 成就
}

interface CategoryProgress {
  category: string;                    // 分类名称
  learned: number;                    // 已学习数量
  total: number;                       // 总数量
}

interface Achievement {
  id: string;                          // 成就ID
  title: string;                       // 成就标题
  description: string;                 // 成就描述
  unlocked_at: string | null;          // 解锁时间
}
```

#### 练习题数据模型

```typescript
interface PracticeQuestion {
  id: string;                          // 题目ID
  function_id: string;                 // 关联公式ID
  type: 'fill_blank' | 'choice' | 'write'; // 题型
  level: '初级' | '中级' | '高级';     // 难度
  question: string;                    // 题目
  data?: any[][];                      // 数据表格（计算题需要）
  template?: string;                   // 模板（填空题需要）
  answer: string;                      // 答案
  explanation: string;                 // 解析
}

interface PracticeRecord {
  id: string;                          // 记录ID
  question_id: string;                 // 题目ID
  user_answer: string;                 // 用户答案
  is_correct: boolean;                 // 是否正确
  answered_at: string;                 // 答题时间
}
```

## 3. 数据流

### 3.1 公式搜索流程

```
用户输入关键词
    │
    ▼
前端：搜索算法（模糊匹配、关键词匹配）
    │
    ▼
前端：筛选（分类、难度、版本、热度）
    │
    ▼
前端：排序（热度、字母、使用频率）
    │
    ▼
前端：渲染搜索结果
```

### 3.2 公式详情流程

```
用户点击公式
    │
    ▼
前端：请求公式详情
    │
    ▼
后端：从JSON文件读取公式数据
    │
    ▼
后端：返回公式数据（IPC）
    │
    ▼
前端：渲染详情页
    │
    ▼
前端：记录学习进度
```

### 3.3 收藏功能流程

```
用户点击收藏
    │
    ▼
前端：调用收藏命令（IPC）
    │
    ▼
后端：更新用户收藏数据（JSON）
    │
    ▼
后端：返回成功状态
    │
    ▼
前端：更新UI状态
```

### 3.4 学习进度流程

```
用户查看公式详情（>10秒）
    │
    ▼
前端：记录查看时长
    │
    ▼
前端：定时检查（每30秒）
    │
    ▼
如果>10秒，调用学习进度命令（IPC）
    │
    ▼
后端：更新用户学习数据（JSON）
    │
    ▼
后端：返回成功状态
    │
    ▼
前端：更新进度显示
```

## 11. 架构决策记录（ADR）

### ADR-001：使用Tauri + Vue作为技术栈

**背景：** 需要开发Windows桌面应用，要求文件大小<50MB，性能优秀。

**决策：** 使用Tauri + Vue。

**理由：** 详见《技术选型报告》

**日期：** 2026-03-28

---

### ADR-002：使用JSON作为数据存储格式

**背景：** 需要存储公式数据、用户数据、练习题数据。

**决策：** 使用JSON格式存储。

**理由：**
1. JSON格式简单，易于读写
2. 前后端都原生支持JSON
3. 文件体积小，易于传输
4. 可以用lowdb轻松管理

**替代方案：**
1. SQLite：功能强大，但需要额外依赖
2. YAML：格式更简洁，但Rust支持不如JSON好

**日期：** 2026-03-28

---

### ADR-003：前端使用Pinia作为状态管理

**背景：** 需要管理应用状态（搜索、收藏、进度等）。

**决策：** 使用Pinia。

**理由：**
1. Pinia是Vue官方推荐的状态管理库
2. 支持TypeScript
3. API简洁，易于使用
4. 支持模块化

**替代方案：**
1. Vuex：Pinia的替代品，但API较复杂
2. 简单状态管理：直接用reactive，但不易维护

**日期：** 2026-03-28
