## 1. 需求背景

### 1.1 痛点问题
业务人员在日常工作中使用Excel时，经常遇到以下问题：

1. **不会写公式** - 不知道用什么公式解决特定问题
2. **写公式太慢** - 查找公式语法和参数耗时
3. **忘记公式用法** - 常用公式忘记参数顺序或使用方法
4. **工作场景脱节** - 学习了公式但不知道如何应用到实际工作中
5. **缺乏个性化指导** - 不知道自己还缺什么，学习路径不清晰

### 1.2 产品定位
一款Excel公式学习与查询工具，帮助业务人员快速查找、学习、对比、练习Excel公式，提供实战案例和个性化学习路径，提升工作效率。

### 1.3 目标用户
- **主要用户**：业务人员（销售、运营、财务、行政等）
- **次要用户**：数据分析师、需要处理Excel数据的工作人员

## 4. 功能详细说明

### REQ-001：公式库浏览

**功能描述：**
用户可以浏览所有Excel公式，按分类查看，区分常用函数和罕见函数。

**分类体系：**

| 大类 | 小类 | 说明 |
|------|------|------|
| 数学与三角 | 常用 | SUM, AVERAGE, COUNT, MAX, MIN 等 |
| | 高级 | SUMIF, SUMIFS, SUMPRODUCT, ROUND 等 |
| 文本 | 基础 | LEFT, RIGHT, MID, LEN, CONCAT 等 |
| | 高级 | TEXT, SUBSTITUTE, FIND, REPLACE 等 |
| 日期与时间 | 基础 | TODAY, NOW, YEAR, MONTH, DAY 等 |
| | 高级 | DATEDIF, EOMONTH, WORKDAY, NETWORKDAYS, DAYS, EDATE 等 |
| 查找与引用 | 基础 | VLOOKUP, HLOOKUP, INDEX, MATCH 等 |
| | 高级 | XLOOKUP, XMATCH, INDIRECT, OFFSET, CHOOSE 等 |
| 逻辑 | 基础 | IF, AND, OR, NOT 等 |
| | 高级 | IFERROR, IFS, SWITCH 等 |
| 财务 | 基础 | PV, FV, PMT, RATE, NPER 等 |
| | 高级 | NPV, IRR, XNPV, XIRR, SLN, DDB, SYD, DB 等 |
| 统计 | 基础 | AVERAGE, MEDIAN, MODE.SNGL 等 |
| | 高级 | STDEV.S, STDEV.P, VAR.S, VAR.P 等 |
| 数据库 | 全部 | DSUM, DAVERAGE, DCOUNT, DGET 等 |
| 信息 | 全部 | ISBLANK, ISERROR, ISTEXT, ISNUMBER 等 |
| 工程 | 全部 | CONVERT, DELTA, GESTEP 等 |
| 兼容性 | 全部 | 兼容旧版本的公式 |
| 动态数组（Office 365） | 全部 | FILTER, SORT, UNIQUE, SEQUENCE, RANDARRAY 等 |

**功能特性：**
- 支持按分类展开/折叠
- 每个分类显示公式数量
- 按热度排序（使用次数多的排在前面）
- 支持字母快速索引（A-Z）
- **常用函数标签**：Top 50常用函数标记"常用"徽章
- **罕见函数标签**：使用频率低的函数标记"罕见"徽章
- **Office 365标签**：仅Office 365支持的函数标记"365"

**验收标准：**
- [ ] 所有分类都能正常展开和折叠
- [ ] 每个分类显示的公式数量准确
- [ ] 点击公式能跳转到详情页
- [ ] 热度排序正常工作
- [ ] 常用/罕见/365标签正确显示

### REQ-002：公式搜索

**功能描述：**
用户可以通过多种方式搜索公式。

**搜索方式：**

| 搜索方式 | 说明 | 示例 |
|---------|------|------|
| 名称搜索 | 按公式名称搜索（模糊匹配） | "VLOOKUP", "SUMIF", "sum" |
| 功能搜索 | 按功能关键词搜索 | "求和", "查找", "日期差", "四舍五入" |
| 分类筛选 | 按分类筛选 | "数学", "文本", "查找" |
| 难度筛选 | 按难度筛选 | "初级", "中级", "高级" |
| 组合搜索 | 名称+分类+功能+难度组合 | 名称包含"SUM"且分类为"数学"且难度为"初级" |

**功能关键词设计：**

#### 数学类关键词
- **求和**：SUM, SUMIF, SUMIFS, SUMPRODUCT, DSUM
- **平均**：AVERAGE, AVERAGEIF, AVERAGEIFS, DAVERAGE
- **计数**：COUNT, COUNTIF, COUNTIFS, COUNTA, COUNTBLANK, DCOUNT
- **最大最小**：MAX, MIN, MAXIFS, MINIFS, LARGE, SMALL
- **四舍五入**：ROUND, ROUNDUP, ROUNDDOWN, INT, TRUNC, MROUND
- **乘幂**：POWER, SQRT, EXP, LN, LOG, LOG10
- **绝对值**：ABS

#### 文本类关键词
- **拼接**：CONCAT, CONCATENATE, TEXTJOIN, &（&符号）
- **提取**：LEFT, RIGHT, MID, LEN
- **查找**：FIND, SEARCH
- **替换**：SUBSTITUTE, REPLACE
- **转换**：TEXT, VALUE, UPPER, LOWER, PROPER, TRIM, CLEAN
- **拆分**：TEXTSPLIT（Office 365）

#### 日期时间类关键词
- **当前**：TODAY, NOW
- **提取**：YEAR, MONTH, DAY, HOUR, MINUTE, SECOND
- **计算**：DATEDIF, EOMONTH, WORKDAY, NETWORKDAYS, DAYS, EDATE
- **格式**：TEXT, DATE, TIME
- **星期**：WEEKDAY, WEEKNUM

#### 查找引用类关键词
- **查找**：VLOOKUP, HLOOKUP, LOOKUP, XLOOKUP
- **灵活查找**：INDEX, MATCH, XMATCH
- **引用**：INDIRECT, OFFSET, CHOOSE
- **行列**：ROW, COLUMN, ROWS, COLUMNS
- **位置**：ADDRESS, HYPERLINK

#### 逻辑类关键词
- **判断**：IF, IFS, SWITCH
- **且或**：AND, OR, NOT
- **错误处理**：IFERROR, ISERROR, ISNA, ISERR

#### 统计类关键词
- **集中趋势**：AVERAGE, MEDIAN, MODE.SNGL
- **离散程度**：STDEV.S, STDEV.P, VAR.S, VAR.P
- **排位**：RANK.EQ, RANK.AVG, PERCENTILE.EXC, PERCENTILE.INC
- **计数**：COUNT, COUNTA, COUNTBLANK, COUNTIF, COUNTIFS
- **频数**：FREQUENCY

#### 财务类关键词
- **货币时间价值**：PV, FV, PMT, RATE, NPER
- **投资评估**：NPV, IRR, XNPV, XIRR
- **折旧**：SLN, DDB, SYD, DB
- **贷款**：PMT, IPMT, PPMT

**搜索规则：**
- 支持模糊搜索（输入"sum"可匹配"SUM", "SUMIF", "SUMIFS"）
- 支持中英文搜索
- 搜索结果实时更新（输入即搜索）

### REQ-003：公式详情

**功能描述：**

```

├─────────────────────────────────┤

│ 分类: 数学→查找                          │
│ 难度: 初级                               │
│ 热度: ★★★★☆                              │
│ 版本: 常用函数                            │
│ 描述: 根据表格首列查找值，返回指定列的值   │
├─────────────────────────────────┤
│ 【语法】                                 │
│ =VLOOKUP(lookup_value, table_array,     │
│   col_index_num, [range_lookup])         │
│                                           │
│ 参数说明：                                │
│ • lookup_value (必需)                    │
│   要查找的值，可以是值、单元格引用或文本   │
│                                           │
│ • table_array (必需)                     │
│   要查找的数据表范围，首列必须包含查找值   │
│                                           │
│ • col_index_num (必需)                   │
│   返回值所在的列号（从1开始计数）         │
│                                           │
│ • range_lookup (可选)                    │
│   查找方式：FALSE=精确匹配，TRUE=近似匹配  │
│   省略时默认为TRUE                         │
├─────────────────────────────────┤
│ 【示例1：基本精确查找】                   │
│ 场景：查找员工张三的工号                   │
│                                           │
│ 数据表:                                   │
│    A      B      C                       │
│ 1  姓名   工号   薪资                    │
│ 2  张三   001    5000                    │
│ 3  李四   002    6000                    │
│ 4  王五   003    7000                    │
│                                           │
│ 公式:                                     │
│ =VLOOKUP("张三", A1:C4, 2, FALSE)        │
│                                           │
│ 结果:                                     │
│ 001                                       │
│                                           │
│ 详细说明：                                │
│ • "张三"：要查找的值                       │
│ • A1:C4：查找范围，首列A列包含姓名         │
│ • 2：返回第2列（工号列）                   │
│ • FALSE：精确匹配                         │
├─────────────────────────────────┤
│ 【示例2：近似查找】                       │
│ 场景：根据分数查找对应等级                 │
│                                           │
│ 数据表:                                   │
│    A      B                              │
│ 1  分数   等级                            │
│ 2  0      F                               │
│ 3  60     D                               │
│ 4  70     C                               │
│ 5  80     B                               │
│ 6  90     A                               │
│                                           │
│ 公式:                                     │
│ =VLOOKUP(75, A1:B6, 2, TRUE)             │
│                                           │
│ 结果:                                     │
│ C                                         │
│                                           │
│ 详细说明：                                │
│ • 75：要查找的分数                         │
│ • A1:B6：查找范围，首列A列包含分数         │
│ • 2：返回第2列（等级列）                   │
│ • TRUE：近似匹配（找到≤75的最大值70）     │
│ • 注意：使用近似查找时，查找范围必须按升序排列│
├─────────────────────────────────┤
│ 【常见错误案例】                           │
│                                           │
│ ❌ 错误1：col_index_num超出范围            │
│ 公式: =VLOOKUP("张三", A1:C4, 5, FALSE)   │
│ 错误: #REF!                               │
│ 原因: 数据表只有3列，但要求返回第5列       │
│ 修正: =VLOOKUP("张三", A1:C4, 3, FALSE)   │
│                                           │
│ ❌ 错误2：range_lookup省略导致近似匹配     │
│ 公式: =VLOOKUP("张三", A1:C4, 2)          │

├─────────────────────────────────┤
│ 【使用建议】                               │
│ • 适用于：查找列在数据表右侧的场景          │
│ • 精确查找：记得加上FALSE参数              │

│ • range_lookup省略时默认为TRUE（近似查找）  │
│ • 查找值区分大小写                         │

├─────────────────────────────────┤
│ 【相关公式】                               │
│ • HLOOKUP - 按行查找                       │
│ • INDEX+MATCH - 灵活查找（任意方向）        │
│ • XLOOKUP - 新版查找函数（Office 365）     │
│ • LOOKUP - 向量查找                        │
├─────────────────────────────────┤
```

### REQ-005：公式对比

**功能描述：**
用户可以对比多个相似公式的区别和使用场景，系统自动给出使用建议。

**对比功能：**
- 最多支持3个公式对比
- 对比项：语法、参数、优缺点、使用场景、性能、Excel版本、限制
- 表格形式展示对比结果
- 支持导出对比结果
- **自动推荐对比**（查看VLOOKUP时，推荐对比INDEX+MATCH、XLOOKUP）
- **使用建议**（自动给出具体场景下推荐用哪个函数）

**对比示例：**

| 对比项 | VLOOKUP | INDEX+MATCH | XLOOKUP |
|--------|---------|-------------|---------|
| **语法复杂度** | 简单 | 中等 | 简单 |
| **查找方向** | 只能从左到右 | 任意方向 | 任意方向 |
| **性能** | 较慢 | 较快 | 最快 |
| **多条件查找** | 需要辅助列 | 可以直接实现 | 可以直接实现 |
| **模糊查找** | 支持 | 需要额外处理 | 支持 |
